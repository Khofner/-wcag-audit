Everything you need to know about the WCAG Audit dashboard

Getting started
An AI-assisted WCAG 2.2 accessibility audit dashboard for development and design teams. It combines AI screenshot analysis, code review with auto-fix, and team collaboration in one place.
Visit console.anthropic.com/settings/keys, sign in, click "Create key", and copy it immediately. It starts with sk-ant-. Paste it into the API Key field in the sidebar.
No. Your key lives only in your browser session. It is never sent to our servers, never written to a database, and clears when you close the tab. API calls go directly from your browser to Anthropic.
We use magic link authentication via Supabase. Enter your work email, receive a link in your inbox, click it, and you're in. No passwords required. Sessions last 8 hours.
Screenshot audit
All 54 WCAG 2.2 criteria, categorized as AI-reviewed (visually detectable), Partial (state-dependent), or Manual (requires live testing). AI-reviewed criteria include alt text, contrast, heading structure, label presence, target sizes, and more.
High — clear evidence in the screenshot. Medium — likely issue, depends on context. Low — pattern-based suggestion that may not apply. Always have a human confirm Low confidence findings.
Code reviewer
Paste code, run review, apply HTML fixes in the editor, click Run Again to re-analyze. Typically 2–3 passes resolve all auto-fixable HTML issues. Remaining findings (CSS, JS, Design, Content) are tagged for your team.
HTML — markup change, auto-fixable. CSS — stylesheet change. JS — behavior change. Design — design decision needed. Content — text/copy change.
Roadmap
Persistent audit history, team compliance dashboard, compliance trend charts, and Jira ticket creation directly from findings. When a Jira ticket closes, the finding status updates automatically.
