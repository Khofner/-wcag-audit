Getting started

This is an AI-assisted WCAG 2.2 accessibility audit dashboard built for development and design teams. It helps you identify, track, and remediate accessibility issues across your product — using AI to analyze screenshots and code, then generate structured findings your team can act on.

Screenshot audit

The screenshot audit covers all 54 WCAG 2.2 criteria, categorized as: AI reviewed (visually detectable), Partial (state-dependent — may need additional screenshots), and Manual (requires live testing with a keyboard or screen reader). AI-reviewed criteria include alt text, contrast, heading structure, label presence, target sizes, sensory instructions, and more.
Keyboard accessibility, caption availability, timing controls, and screen reader behavior require live testing — these are flagged as Manual in the findings. For best coverage, upload multiple screenshots showing different states: default, hover, focused, error, and disabled.
High — clear evidence in the screenshot. Medium — likely issue but depends on context not fully visible. Low — pattern-based suggestion that may not apply. Low confidence findings are shown with a best-guess and a review flag — always have a human confirm these.

Code reviewer

Paste your code and run a review. Apply HTML fixes directly in the editor with the Auto-fix button — the editor updates in place. Click Run Again to re-analyze the updated code. Typically 2–3 passes resolve all auto-fixable HTML issues. Remaining findings (CSS, JS, Design, Content) are tagged for your team.
HTML — markup change, auto-fixable in the editor. CSS — stylesheet change, needs your frontend dev. JS — behavior change, needs your frontend dev. Design — requires a design decision (color, layout, spacing). Content — text or copy change needed.
Ignored findings move to a collapsed Ignored section and won't reappear on subsequent Run Again passes. They appear in the exported report under "Reviewed — intentionally not addressed", which gives stakeholders full visibility into what was reviewed and deliberately skipped. You can un-ignore at any time.
